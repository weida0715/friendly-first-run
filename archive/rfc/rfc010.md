## RFC-010: Model Rankings, Model Detail, Model Library, and Favorites

### Status

Proposed

### Summary

Implement model result viewing and comparison. Users should inspect generated models, sort by metrics, view details, and favorite models.

### Problem Statement

Experiments generate multiple model artifacts. Users need a structured way to compare results, inspect parameters, view metrics, and save useful models.

### Goals

- Implement model detail page.
- Implement model rankings page.
- Implement sorting and filtering.
- Implement model library.
- Implement favorite/unfavorite behavior.
- Protect private model access.

### Non-Goals

- No experiment training logic here.
- No Public Hub exposure here beyond access-safe model eligibility.

### Implementation Plan

#### Stage 1: Implement model detail api and page

Implement ModelController and ModelDetailView for viewing a single generated model artifact. Display Model parameters, parent Experiment, selected Blueprint, performance metrics, confusion metrics, evaluation metrics, run metadata, and log summary. Use AccessControlService to verify that users can only view owned, accessible, or public-eligible Models. Add links from ExperimentDetailView to related ModelDetailView pages.

Refs: ModelController, ModelDetailView, ExperimentDetailView, AccessControlService, Model, Experiment, Blueprint

#### Stage 2: Implement rankings sorting and metric

Implement ModelsRankingsController and ModelsRankingsView. Add ranked model table with sorting and filtering by Sharpe ratio, accuracy, precision, recall, false positive rate, AUC, maximum drawdown, win rate, experiment, Blueprint, owner, and creation date. Add pagination, metric formatting, status badges, and links to ModelDetailView. Ensure only accessible models are returned for authenticated users.

Refs: ModelsRankingsController, ModelsRankingsView, AccessControlService, Model

#### Stage 3: Implement model library and favorite workflow

Implement ModelsLibraryController, ModelsLibraryView, and FavoriteModel persistence. Add Owned and Favorited tabs, favorite and unfavorite actions, model cards or table rows, and immediate UI feedback after favorite removal. Ensure FavoriteModel links User and Model through a composite relationship and that private model access remains protected through AccessControlService.

Refs: ModelsLibraryController, ModelsLibraryView, FavoriteModel, ModelController, AccessControlService

#### Stage 4: Add model detail metrics and access tests

Add tests for ModelController and ModelDetailView. Verify model detail retrieval, parameter display, Experiment linkage, Blueprint linkage, confusion metric display, evaluation metric display, log summary display, owned model access, public-eligible model access, and private model access rejection for unauthorized users.

Tests: ModelController, ModelDetailView, AccessControlService, metrics

#### Stage 5: Add ranking filtering library and favorite tests

Add tests for ModelsRankingsController, ModelsRankingsView, ModelsLibraryController, ModelsLibraryView, and FavoriteModel. Verify sorting by Sharpe ratio, accuracy, precision, recall, false positive rate, AUC, maximum drawdown, and win rate. Verify filtering by Experiment and Blueprint. Verify owned model listing, favorited model listing, favorite creation, favorite removal, duplicate favorite prevention, and unauthorized private model exclusion.

Tests: rankings, filtering, model library, favorites

### Pseudocode

```python
def get_ranked_models(user, sort="sharpe", filters=None):
    query = model_repo.query_accessible(user.id)

    query = apply_filters(query, filters)
    query = apply_sort(query, sort)

    return paginate(query)
```

```python
def toggle_favorite_model(user, model_id):
    model = model_repo.get_by_id(model_id)

    if not access_control.can_view_model(user, model):
        raise ForbiddenError()

    if favorite_repo.exists(user.id, model_id):
        favorite_repo.delete(user.id, model_id)
        return {"favorited": False}

    favorite_repo.create(user.id, model_id)
    return {"favorited": True}
```
