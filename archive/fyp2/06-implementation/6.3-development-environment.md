### 6.3 Development Environment

The development environment was prepared to support the layered implementation direction described in Chapter 5 and the BTCUSDT-focused research scope defined in Chapters 1 to 3. The environment separates frontend, backend, database, and testing concerns so that the web boundary, controller layer, business logic, domain entities, and persistence layer can be developed and verified independently.

#### 6.3.1 Programming Languages Used

The implementation uses different languages according to the responsibility of each layer. TypeScript and JavaScript support the web boundary, Python supports the backend control and business logic layers, while SQL and styling languages support persistence verification and user-interface presentation.

| Language | Usage in BEE Implementation |
| --- | --- |
| TypeScript / JavaScript | Used for the Next.js frontend, React components, route files, frontend API client, authentication provider, validators, and Jest-based frontend tests. |
| Python 3.11 | Used for the Flask backend, controllers, services, validators, repositories, domain models, value objects, database session handling, and pytest-based backend tests. |
| SQL | Used indirectly through SQLAlchemy ORM mappings and Alembic migrations, and directly for validating PostgreSQL schema constraints, table names, column names, primary keys, foreign keys, and enum values. |
| HTML / JSX | Used through React and Next.js page/view components to define the web interface structure. |
| CSS | Used through Tailwind CSS utility classes and shadcn-style component styling to implement the prototype-aligned interface. |
| Markdown | Used for repository documentation, report chapter drafting, implementation notes, and archived RFC/reference documents. |

#### 6.3.2 Frameworks and Libraries

The selected frameworks and libraries reflect the architecture choices justified in the earlier design chapter: a Next.js frontend for the web interface, a Flask backend for REST-oriented control logic, and SQLAlchemy with PostgreSQL for structured persistence. Additional libraries support authentication, validation, testing, market-data integration, and future experiment execution workflows without changing the layered separation of concerns.

| Layer / Concern | Technology | Purpose |
| --- | --- | --- |
| Frontend Framework | Next.js 15 | Provides the React-based application framework, App Router structure, page routing, layout handling, and frontend build pipeline. |
| Frontend Library | React 19 / React DOM | Used to build the component-based Boundary-layer views and reusable UI components. |
| Frontend Language Tooling | TypeScript | Provides static typing for frontend views, components, API client functions, route configuration, and validation helpers. |
| UI Styling | Tailwind CSS | Provides utility-first styling for responsive layouts, spacing, typography, and component composition. |
| UI Component Pattern | shadcn-style components, Radix Slot, class-variance-authority, tailwind-merge | Supports reusable form, layout, status, table, badge, and dialog components while keeping the UI consistent with the prototype. |
| Icons | Lucide React | Provides icon components used in navigation, dashboard actions, buttons, and status-oriented interface elements. |
| Charting | TradingView Lightweight Charts / lightweight-charts | Prepared for BTCUSDT price visualization in dashboard and experiment-related views. |
| Backend Framework | Flask | Provides the REST API controller layer, route registration, request handling, and application factory structure. |
| Backend Authentication Support | Flask-Login | Included for session-based authentication support in the backend environment. |
| Backend Security Support | Flask-WTF | Included for CSRF-related security support for state-changing requests. |
| Password and Request Utilities | Werkzeug | Supports password hashing utilities and Flask-compatible request/response behaviours. |
| Configuration Loading | python-dotenv | Supports environment-variable loading for local development configuration. |
| Data Access | SQLAlchemy | Implements ORM mappings, database sessions, and repository-backed persistence. |
| Database Migration | Alembic | Manages database schema migrations for the strict ERD-aligned PostgreSQL schema. |
| Database | PostgreSQL | Used as the runtime relational database target through `DATABASE_URL`, matching the strict persistence scope. |
| Market Data Integration | Binance Connector | Included for BTCUSDT market-data retrieval in the market-data integration boundary. |
| Data Processing | pandas, NumPy, scikit-learn | Included for data manipulation, numerical computation, and later experiment/model execution logic. |
| Queue and Background Processing | Redis, RQ | Included for background job and queue infrastructure boundaries used by experiment execution planning. |
| Frontend Testing | Jest, React Testing Library, jest-dom, user-event, ts-jest | Used for component rendering, route, navigation, authentication guard, wizard, and UI behaviour tests. |
| Backend Testing | pytest, pytest-cov | Used for backend controller, service, repository, domain, database, and validation tests. |

#### 6.3.4 Operating System Used

Development was performed in a local Linux environment using kernel version 6.12. The repository was developed with separate frontend and backend workspaces, allowing the Next.js frontend and Flask backend to run as independent local services during development and verification.

| Environment Item | Configuration |
| --- | --- |
| Operating system | Linux 6.12 development environment |
| Frontend server | Next.js development server, typically served from `http://localhost:3000` |
| Backend server | Flask application server, typically served from `http://localhost:5000` |
| Backend API health endpoint | `GET /api/health` |
| Frontend backend proxy path | `/api/backend/*`, forwarded to the Flask backend according to `frontend/next.config.ts` |
| Database target | PostgreSQL through the `DATABASE_URL` environment variable |
| Project root | `/mnt/SSD/MMU Stuff/FYP/Bitcoin-Experimental-Engine` |
